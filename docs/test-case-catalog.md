# My-eBookStore 自动化测试用例与预期结果

## 1. 说明

- 当前完整测试套件共收集 24 个用例（参数化展开后）。
- 黑盒测试通过 HTTP 请求验证输入和响应；灰盒测试在 HTTP 请求后查询 SQL Server 验证业务状态；白盒测试直接调用内部函数。
- 集成测试要求 `EBOOKSTORE_ENV_FILE=.env.test`，数据库名称以 `_Test` 结尾，并已执行全部迁移和 `99_test_seed.sql`。
- 每个用例显示为 `PASSED` 表示实际结果符合预期；完整成功输出为 `24 passed`。

## 8. 执行及结果判断

```powershell
$env:EBOOKSTORE_ENV_FILE = ".env.test"
python -m pytest -q
```

完整通过的预期终端摘要：

```text
24 passed
```

分类执行：

```powershell
python -m pytest -m "not integration" -v
python -m pytest -m "integration and not performance" -v
python -m pytest -m performance -v
```

- `PASSED`：实际结果符合本文件中的预期输出。
- `FAILED`：断言未满足，下方 traceback 给出实际值和失败位置。
- `ERROR`：测试环境、夹具或数据库初始化失败，不能视为业务用例失败。
- 第三方 `StarletteDeprecationWarning` 不影响当前测试结论，但应在依赖升级时处理。
